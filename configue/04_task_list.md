# 프로젝트 개발 로드맵 (Task List)

이 문서는 앞선 기획안과 규칙들을 바탕으로 한 프로젝트의 세부 체크리스트입니다. AI 에이전트(Flash)는 개발을 진행하면서 이 Task 스텝을 기준으로 작업 진척도를 관리하고, 마일스톤(Phase)이 끝날 때마다 GitHub 커밋을 제안해야 합니다.

## Phase 1: 기반 인프라 세팅 및 사용자 인증 (Setup & Auth)
- [x] Next.js (프론트엔드) 스캐폴딩 및 Tailwind CSS, UI 라이브러리(shadcn 등) 설치
- [x] FastAPI (백엔드) 통합 환경 세팅 (CORS 처리 및 REST 라우팅 구조 확립)
- [x] Supabase 프로젝트 생성 및 DB 구조 설계 (유저 정보, 템플릿 목록, 히스토리 테이블)
- [x] 사용자 회원가입/로그인 (Supabase Auth) 연동 및 프론트/백엔드 토큰 검증
- [x] Supabase Storage 세팅 완료 (원본 `.pptx` 템플릿 파일 업로드용 버킷)

## Phase 2: 코어 모듈 구현 (AI 리서치 + 파워포인트 생성)
- [x] **AI 딥리서치 모듈:** Gemini API 연계. 주제와 "생성 분량(기본 8장, 최대 15장)" 제어 파라미터를 받아 슬라이드 내용을 분할한 JSON 데이터 도출
- [ ] **PPT 조립 모듈:** `python-pptx`를 이용해 Storage의 템플릿 파일을 받아 슬라이드를 마스터 구조에 따라 복제
- [ ] 추출된 JSON 데이터를 각 슬라이드(제목, 본문 Placeholder) 구조에 텍스트로 치환 매핑
- [ ] 텍스트 처리 과정 중 코드 구조로 '사용자 폰트 이름' 속성 덮어씌우기 (단순화된 폰트 적용)
- [ ] 🏆 **메타데이터 조작:** `python-pptx`의 core properties 기능을 사용하여 완성된 파일에 \[생성일, 사용 템플릿, 폰트\] 꼬리표 값 숨겨서 박아넣기 성공

## Phase 3: 비동기 큐잉 시스템 도입 (대규모 트래픽 대비)
- [ ] Redis 서버 셋업 및 Celery Task Worker 연동 (또는 상응하는 파이썬 백그라운드 큐 환경 구축)
- [ ] PPT 생성 코어 모듈(Phase 2 전체)을 Celery 백그라운드 태스크로 이관
- [ ] 50명 동시 접속에 대응하는 작업 대기열(Queue) 로드 밸런싱 모니터링 환경 설정
- [ ] 상태 동기화 처리 (FastAPI - 클라이언트 간 작업 ID를 통한 Polling 혹은 Websocket 로딩 진행률 반영)

## Phase 4: 프론트엔드 통합 연동 및 UI 컴포넌트 개발
- [ ] 메인 대시보드 플로우 구성 (주제 입력 UI + 장수 조절 슬라이더 + [폰트/템플릿 선택] 드롭다운 모델 연동)
- [ ] 동적 생성 로딩/진행률 스피너 및 알림 컴포넌트("AI가 문서를 렌더링하고 있습니다..." 등) 구현
- [ ] 결과물 `.pptx` 다운로드 액션/버튼과 스토리지 다운로드 엔드포인트 연동
- [ ] **마이페이지(기록):** DB를 조회하여 사용자가 생성했던 'PPT 작업 히스토리' 표 UI 렌더링 및 재다운로드 버튼 구현

## Phase 5: 최종 디버깅 루프 가동 및 UAT (사용자 인수 테스트)
- [ ] 에러 로그(Context Memory, `.ai_error_log.md`)가 정상적으로 기록되고 복구 피드백 루프가 제동하는지 고의적인 버그 상황에서 테스트
- [ ] 타겟 환경(폰트 미설치/설치 PC)에서의 파일 열람 레이아웃 시각적 검증
- [ ] 전체 서비스 실 데이터 기반 통합 테스트(E2E) 완료 및 개발 완료 브리핑
