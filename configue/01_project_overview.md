# 프로젝트 개요 및 핵심 요구사항 (PPT Agent)

## 1. 프로젝트 목표
회사의 PPT 템플릿 디자인과 전용 폰트를 활용하여, 제미나이(Gemini) AI의 딥리서치 결과를 바탕으로 자동으로 파워포인트 슬라이드를 생성해주는 웹 기반 AI 에이전트 서비스.

## 2. 핵심 타겟 페르소나 및 유즈케이스
- **타겟:** 사내 워크숍, 강의 준비자, 기획자 등 빠르고 일관된 양식의 초안 PPT가 필요한 사용자.
- **주요 흐름:** 주제 입력 -> 템플릿/폰트 선택 -> [슬라이드 장수 조절] -> AI 리서치 및 구조화 -> PPT 생성 -> 다운로드

## 3. 상세 기능 요구사항 (MVP 기준)

**1) AI 리서치 및 내용 구조화**
- Gemini API 및 MCP(웹 검색 등 딥리서치 도구) 연동.
- 사용자가 입력한 주제를 기반으로 정확히 **지정된 장수(기본 8장, 최대 15장 내외 조절 가능)**로 메인 메시지와 본문을 구조화하여 JSON 형태로 추출.

**2) PPT 템플릿 및 폰트 적용**
- **템플릿:** 사용자가 업로드한 사내 양식(`.pptx`)의 슬라이드 마스터 레이아웃 기준 복제.
- **폰트:** 사용자가 지정한 폰트명(예: '나눔바른고딕')을 텍스트 속성에 강제 지정 (단순화 방식 채택하여 서버 부하 및 복잡도 감소).

**3) 메타데이터 및 문서 내역 관리 (핵심 구현)**
- **숨겨진 파일 속성:** 생성된 `.pptx` 파일의 문서 속성(만든 이, 태그, 설명 등)에 생성일시, 사용 폰트, 사용 템플릿 정보를 서버측 코드로 삽입(메타데이터 기록).
- **웹 대시보드 히스토리:** 사용자의 마이페이지에 생성 이력(날짜, 주제, 설정, 다운로드 링크 기록)을 표 형태로 제공하여 파일 유실 시 재다운로드 지원.

**4) 웹 서비스 기본 요구사항**
- 회원가입 및 로그인 (회원 관리).
- 사용자 지정 폰트 이름 및 PPT 디자인 설정 환경(DB) 저장.
- 다수의 템플릿과 폰트를 드롭다운 옵션으로 **교차 생성** 지원.

**5) 트래픽 및 비동기 시스템**
- 약 50명의 동시 생성을 안정적으로 처리.
- 제미나이 리서치 및 파일 렌더링 병목 방지를 위해 **비동기 작업 큐 (Task Queue)** 필수 도입.
- 프론트엔드에서는 로딩 프로그레스(예: "PPT를 구조화 중입니다...") 제공.
